１．CSS詳細度とは
	・・・どのCSSプロパティが要素に最も関係あるか（適用されるか）ブラウザが決定する方法。
	　　　ざっくり言うと、「スタイルが重複した時、どのスタイルを優先するか」の優先度を評	      	　　　価したものである。
	      詳細度が高ければ高いほど優先される。
	　　　
	　　　詳細度はa,b,cのような形で表される（c→b→a→例外2→例外1の順に詳細度が高い）。

	　　　全称セレクタ(*)	                  詳細度   0
	　　　疑似要素(::beforeなど)              詳細度 c=1
	　　　要素セレクタ(divなど)　	          詳細度 c=1
	　　　属性セレクタ([type="submit"]など)   詳細度 b=1
	　　　疑似クラス　　　　　　　　　　　　　詳細度 b=1
	　　　classセレクタ			　詳細度 b=1
	　　　idセレクタ			　詳細度 a=1
	　　　インラインスタイル(style="")	　例外  2
	　　　!important			　例外  1
	　(例) *                 A=0 B=0 C=0  詳細度 =     0
	       div               A=0 B=0 C=1  詳細度 =     1
               ul li             A=0 B=0 C=2  詳細度 =     2
	       ul ol+li          A=0 B=0 C=3  詳細度 =     3
	       .sample div       A=0 B=1 C=1  詳細度 =   1,1
	       .sample .item     A=0 B=2 C=0  詳細度 =   2,0
	       #sample           A=1 B=0 C=0  詳細度 = 1,0,0
	       #sample.sample    A=1 B=1 C=0  詳細度 = 1,1,0

2.スタイルの継承とは
	・・・プロパティには親要素に指定した値が子要素に継承されるものとされないものがある。
	　　　例えば親要素に文字色を指定した場合、子要素に文字色を指定しなくても継承される。
	　　　
	　　　一方、border,margin,padding,背景画像などは継承されない。
	　　　但し、ほぼ全てのプロパティには親要素の値を強制的に継承する「inherit」という値を	　　　	　　　指定することができる。
	       
	      親要素から子要素へ文字サイズなどの値が継承される場合、pt・pxなどの単位の場合に	　　　	　　　はそのままの値が継承されるが、他の要素サイズとの関係で算出される%・em・exなどの	　　　	　　　相対的な単位の場合には、 親要素と子要素の値を掛け合わせた結果の値が継承される。

	　　　例えば、以下のように親要素と子要素のそれぞれに文字サイズ90%を指定した場合、子め	　　　	　　　要素である<p>要素の文字サイズは90%ではなく、90%×90%＝81%となる。

3.marginの相殺とは
	・・・要素が前後に並び、その各々にmarginが設定される時、marginの相殺という現象が起こ	　　　	　　　る。

	　　　１、margin指定が複数並ぶと1つしか適用されない
	　　　　→要素１のmargin-bottomと、その後ろの要素２のmargin-topが並ぶと、値が大きい方	　　　　　	　のmarginが適用される。
	　　　
	　　　２、親要素ともmarginの相殺は起こる
	　　　　→例えばある子要素のmargin-topが40px,その子要素の親要素のmargin-topが10pxの時	　　　　　	　、この親要素の前にある要素とのmarginには子要素からの40pxのmarginが適用され	　　　　　	　る。
	　　　
	　　　つまり、「marginが並ぶとそのうちの１つしか適用されない」、及び「要素間余白が最	　　　	　　　も大きくなるmarginが適用される」。

	　　　floatなどの回り込みの指定の間違いや横幅の合計の間違いなどによって生じる。

8.floatの３つの解除方法
	・・・floatを使用した場合、指定した要素から子要素が認識出来なくなるため、親要素は高さ	　　　	　　　を保持出来なくなる。
	　　　以下はこれを回避するための方法。
	　　　
	　　　１．clear:both;を利用する
	　　　・・・回り込みを解除したい要素に指定する。

	　　　２．親要素に高さを指定する
	　　　・・・ナビゲージョンなど高さの増減がない場合に使用する。

	　　　３．親要素にoverflow:hidden;を指定する
	　　　・・・子要素が全てfloat指定している場合、親要素にoverflow:hidden;を指定すると親	　　　　　　	　　要素が高さを保持出来る。heightを指定する場合と違い、コンテンツが増減して	　　　　　　	　　も高さを可変させることができる。

9.見出しタグの使い分け
	・・・見出しの使い方にはルールがあり、これを守ることで読み手が理解し易く、検索エンジ	　　　	　　　ンにも評価され易い記事を書くことが出来る。

	　　　１．h1～h6を使う順番を守る
	　　　→基本的にh1見出しから順番に使う。

	　　　２．h1タグの使用は1回だけにする
　　	　　　→h1タグは最も大きな見出しで、ウェブページの主題を表すのに一度だけ使うのが理想	　　　　	的である。

	　　　３．h2～h6は何回使ってもＯＫ！
	　　　
	　　　４．記事をいくつかのセクションに分け、その見出しとしてh2～h6タグを使う。

	　　　５．基本的に画像をh2～h6タグで挟むのはNG。
	　　　　　但し、サイトタイトル画像にh1タグを使う場合がある。

	　　　６．「この部分の文字を大きくしたいから」「h3タグの見た目にしたいから」という理	　　　　　	　由で、見出しタグを使えば記事の論理構成が崩れてしまう。		　	　　	　　　　　h1〜h6タグは強調のためではなく「見出し」のために使う。

10.box-sizing ; border-box;
	・・・box-sizingプロパティとは要素の幅と高さの中にpaddingとborderを含めるかどうかを設	　　　	　　　定できるプロパティ。
	　　　border-boxを値に取ると、幅や高さを設定する時、paddingとborderを幅と高さに含める	　　　	      ことになる。
	　　　
	      また、content-boxは初期値であり、paddingとborderを幅と高さに含めない。
	　　　
	　　　inheritを値に取ると、親要素のborder-boxの値を引き継ぐ。	

11.marginとpaddingの違い
	・・・margin  :要素の外側の余白
	　　　padding :要素の内側の余白

12.articleタグとsectionタグの違い
	・・・articleタグ :同じ内容を別ページで再利用する際に各々の個所で使用する。
	　　		　 また、1つの記事として成立するものはarticleタグを使用することが可能　			　　。	　　　

	　　　sectionタグ :説明している内容ごとに区切る時に使用する。
	　　　（例）「猫とは・・・」「犬とは・・・」「ウサギとは・・・」は全て内容ごとに完結	　　　　　　         しているのでsectionタグで囲むことが可能。										
13.サイズの単位 px, % ,em, remの違い
	・・・px  :サイズを絶対値指定
	　　　%   :サイズを相対値指定（親要素基準）
	　　　em  :サイズを相対値指定（親要素基準）、100%=1emに相当する。
	　　　rem :サイズを相対値指定（ルート要素基準）、root emの略。ルート要素(htmlセレクタ		   )を元にしたサイズ指定法。
	　　　　　 （例）html { font-size:20px; }
	    		 p { font-size:10px; }
			 p .test { font-size:2rem; }
		　　　　→p .testのフォントサイズは40pxになる。

14.max-width(max-height), min-width(min-height), vw, vhの違い
	・・・max-width (max-height):<div>や<p>など、内容領域における幅（高さ）の最大値を指定	　　　　　　		　　 	     するプロパティ。 
	　　　min-width (min-height):<div>や<p>など、内容領域における幅（高さ）の最小値を指定	　　　　　　		　　 　　　　するプロパティ。
	　　　vw		    :「viewport width」の略。画面幅に対する相対値。100が基準				      値。
	      vh	  	    :「viewport height」の略。画面高さに対する相対値。100が基				      準値。

	　　　●viewportとは画面サイズのこと。

15.色指定（カラーネーム、16進数、rgba、hsla）の違い
	・・・カラーネーム :redやblue,greenといったカラーネームで指定。

	　　　16進数       :＃で始まる6桁のカラーコードで指定。16進数でい表記。

	　　　rgba　　　　 :rgba(255 ,0 ,0 ,0.5)といったように指定。赤、緑、青、透明度の順に			　　　　　　赤～青までは0～255までの値で、透明度は0（完全に透明)～1（完全に
			　　不透明)までの値で指定。

	　　　hsla	　 :hue(色相),saturation(彩度),lightness(輝度),alphaで色を指定。
	　　		　　hueは0～360°の角度の値で指定。
			　　saturation, lightnessは0～100%の割合で指定。
	  		    alphaは0(完全に透明)～1(完全に不透明)までの値で指定。

17.相対パス、絶対パスの違い
	・・・相対パス：現在地から相対的にファイル等の場所を指定。
	　　　（例）CSS/style.css

　	　　　絶対パス：装置内の上層から特定のファイルなどの場所を指定。
	　　　（例）C:\Lione\・・・\html\html5_css3_level2.txt
